# 安裝後 Acceptance Test 包

> buyer 安裝完 LINE bot 後,跑這份測試包(預計 15 分鐘),把問題截圖貼給「**安裝 Claude**」(buyer 自己的 Claude),Claude 會照 [修正 Playbook](#修正-playbook) 修。

---

## 怎麼用

**buyer 操作:**
1. 在 LINE 上依序傳下方測試題
2. 預期行為對 → 打 ✓
3. 行為不對 → 截圖,加上「題號 + 預期」貼給 buyer 自己的 Claude
4. Claude 會根據下方 Playbook 修

**Claude(buyer 的)操作:**
1. 看到使用者貼測試結果
2. 對照 Playbook 找對應修法
3. 改設定 / system prompt / 工具,重新部署
4. 回 buyer 「再測一次題號 X」

---

## 測試題

### A. 基礎對話(3 題)

**A1. 自我介紹**
- 傳:「你是誰?」
- 預期:回「我是你的 AI 秘書」,提到 Claude 模型,不冒稱別人

**A2. 中文回應**
- 傳:「今天心情不好」
- 預期:繁體中文回應,不混簡體

**A3. 時間概念**
- 傳:「現在幾點?」
- 預期:報出當下台北時間(±5 分鐘內準確)

---

### B. 日期 / 時間理解(4 題,2026-05-18 後常踩)

**B1. 今天日期**
- 傳:「今天幾月幾號星期幾?」
- 預期:正確報出(今天 = 系統時間)

**B2. 「昨天」「明天」**
- 傳:「明天星期幾?」
- 預期:今天 +1 的星期

**B3. 「上週」定義 ⚠️ 易錯**
- 傳:「上週是幾號到幾號?」
- 預期:**上週一到上週日**(ISO 週,不是「今天-7 到今天」)
  例:今天 2026-05-18(週一),上週 = 5/11(一) ~ 5/17(日)
- ❌ 失敗案例:bot 答「5/12~5/18」← 算成滑動窗口

**B4. 「最近 7 天」**
- 傳:「最近 7 天我做了什麼?」
- 預期:滑動窗口(今天-6 到今天),跟「上週」要分清楚

---

### C. Notion 讀取(5 題)

**C1. 今日計畫讀取**
- 傳:「我今天有什麼事?」
- 預期:讀「今日計畫」頁面,列出今天的事項

**C2. 上月評等查詢**
- 傳:「我 4 月 25 號評等如何?」
- 預期:找到「工作記錄:26/04」頁面,讀該天內容

**C3. 長頁面尾端讀取 + 跨日內容分組 ⚠️ 易錯**
- 傳:「上週做了哪些事?」(預期 bot 讀整週,每天分開回答)
- 預期:
  - 標題用 ISO 週(週一~週日),不是「今天-7 到今天」
  - 每天的內容歸到對的 heading(不會把 5/11 的內容當成 5/12)
  - 若某天頁面真的沒寫,bot 老實說「找不到」(不編造)
- ❌ 失敗案例 1:跨日混合 — 「5/12」內容含其實是 5/11 的事
- ❌ 失敗案例 2:長頁面尾端被截 — bot 答「5/X 讀取到的頁面沒顯示」← 沒分頁讀
- ❌ 失敗案例 3:沒寫的日期亂編 — buyer 那天 Notion 沒寫,bot 卻寫出細節

**C4. 搜尋頁面**
- 傳:「找一下我關於『會議記錄』的記錄」
- 預期:用 search_notion 找到對應頁,讀內容

**C5. 不存在的查詢**
- 傳:「3 月 17 日我吃什麼午餐?」(若無此記錄)
- 預期:老實回「找不到」,**不編造**

---

### D. Notion 寫入 / 修改(6 題)

**D1. 基本加入**
- 傳:「幫我加 14:00 看牙醫」
- 預期:add_to_today 呼叫,Notion 上出現新項目,reply 含位置資訊
- 檢查:Notion 上「13:XX」跟「14:00」之間插入

**D2. 純筆記**
- 傳:「記下:研究 prompt caching 機制」(無時間)
- 預期:add_to_today 用 note 類型,插今日區段末尾

**D3. 單筆修改**
- 傳:「把 14:00 看牙醫改成 14:30」
- 預期:update_block 直接呼叫,單筆改完

**D4. 單筆刪除**
- 傳:「刪掉 14:30 看牙醫」
- 預期:delete_block 直接呼叫,刪完告訴使用者 trash 30 天可救回

**D5. 批次修改 + 確認流程 ⚠️ 易錯**
- 傳:「把今天所有排程往後挪 30 分」
- 預期:propose_batch_action 提案,bot reply 完整清單 + [✓ 確認][✗ 取消] 按鈕,90 秒過期
- 接著傳「確認」→ 全部執行
- ❌ 失敗案例:bot 編造 block_id 全 404,或 bot 寫「以上⋯」清單缺失

**D6. 模糊請求不亂寫 ⚠️ 易錯**
- 傳:「測試一下」「看看」「試試看」
- 預期:**bot 直接對話回應,不呼叫任何寫入工具**
- ❌ 失敗案例:bot 把對話歷史中「之前提過要加的事」自動寫進 Notion

---

### E. 撤回 / 安全網(4 題)

**E1. 撤回單筆**
- 接 D1 之後 → 傳「撤回剛才那筆」
- 預期:從 Notion 刪掉那筆,reply 確認

**E2. 列出今日寫入**
- 傳:「今日寫入」
- 預期:列出今天 bot 動過的清單(倒數編號)

**E3. /狀態 指令**
- 傳:「狀態」
- 預期:顯示本月 API 用量、今日對話數、今日寫入、近 7 天錯誤

**E4. /help 指令**
- 傳:「/help」
- 預期:列出所有 bot 指令

---

### F. 對話歷史不污染當前(2 題)

**F1. 不根據歷史補做 ⚠️ 嚴重 bug**
- 先傳:「等下我要記得加 14:00 開會」(但不要正式要求加)
- 隔幾分鐘傳:「電腦版測試一下」
- 預期:**bot 完全不會自動加 14:00 開會**
- ❌ 失敗案例:bot 把對話歷史中提過的事項自動寫進 Notion

**F2. 不綁 batch ⚠️ 嚴重 bug**
- 先傳:「全部往後 30 分」(bot 提案後**不要確認**,等 90 秒過期)
- 再傳:「加 17:30 處理影片」
- 預期:bot **只加 17:30 處理影片**,不會把「全部往後 30 分」綁進新 batch

---

### G. 語音訊息(3 題)

**G1. 基本語音轉文字**
- 傳語音(說一句話):「今天天氣不錯」
- 預期:bot 回覆**開頭先 quote**「你說:『今天天氣不錯』」,再正式回應

**G2. 語音含寫入指令**
- 傳語音:「幫我加下午三點寫稿」
- 預期:正確抓到時間 + 內容,add_to_today 執行

**G3. 雜訊或不清楚的語音**
- 故意小聲或含糊念
- 預期:Whisper 回的文字怪怪的時,bot 照實 quote 不偷改,問使用者「你是說...嗎?」

---

### H. 邊界情境(3 題)

**H1. 連續快速 5 則**
- 1 分鐘內快速傳 5 則訊息
- 預期:每則都回應,不會卡死

**H2. 超長訊息**
- 傳一則 1000+ 字的訊息
- 預期:bot 回應(可能慢),不會炸

**H3. 純表情或貼圖**
- 傳一個 emoji 或貼圖
- 預期:不報錯(可能不回應,但 bot 不會掛)

---

## 修正 Playbook(給 buyer 的 Claude 看)

> 使用者貼測試結果給你,你照下表找對應修法。

| 測試題 | 失敗症狀 | 修法 |
|---|---|---|
| B3 | 「上週」算成「今天-7 到今天」 | system prompt 加 ISO 週定義,參考 [INSTALL-GOTCHAS.md 坑 #18 系列](INSTALL-GOTCHAS.md) |
| C3 | 長頁面尾端讀不到 | `read_notion_page` 改成分頁讀(`has_more` + `start_cursor`),字數上限 30000 |
| D5 失敗(404) | Claude 編造 block_id | `read_notion_page` 每行加 `[block:xxx]`,system prompt 強調「block_id 必從標記抓」(坑 #21) |
| D5 失敗(清單缺) | bot 寫「以上...」沒清單 | line.ts 直接從 KV 撈 pending 組訊息,不靠 Claude 轉述(坑 #22) |
| D6 失敗 | 模糊請求亂寫 Notion | system prompt 加「模糊請求 → 不呼叫寫入工具」(坑 #18) |
| F1 失敗 | 根據歷史補做 | system prompt 加「只執行當前訊息的明確要求」(坑 #18) |
| F2 失敗 | 自動 batch 綁歷史 | system prompt 加「禁止把當前+歷史請求綁 batch」 |
| G1 失敗 | 沒 quote 使用者語音 | system prompt 加「語音訊息回應規則,開頭必 quote」 |
| H1 失敗(429) | 短時間 rate limit | 看 system prompt 大小,shared memory > 15K token 要砍(坑 #23) |
| 普遍慢/卡 | 5 秒 LINE timeout | 必須用 Ack+Push 架構(坑 #24) |

## 後記

這份測試包**會隨 bot 改版擴充**。每次 bot 出新功能 / 修新 bug,對應加一題進來。

設計原則:**每個踩過的雷,都對應一題 acceptance test**,確保不會 regression。
